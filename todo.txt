# Penguin OS - Development Roadmap & TODO

This file contains improvements, fixes, and enhancements to make Penguin OS more robust,
production-ready, and useful for practitioners and industry use cases.

## CRITICAL SECURITY ISSUES

### Authentication & Authorization
- [ ] Implement refresh token mechanism to improve JWT security (currently only access tokens)
- [ ] Add password reset functionality via email with secure token expiration
- [ ] Implement email verification for new user registrations
- [ ] Add two-factor authentication (2FA) support using TOTP
- [ ] Remove hardcoded JWT_SECRET from docker-compose.yml (use Docker secrets or env files)
- [ ] Implement CSRF protection for state-changing operations
- [ ] Add rate limiting per user/account (not just per IP) to prevent abuse
- [ ] Implement account lockout after multiple failed login attempts
- [ ] Add session management UI for users to view/revoke active sessions
- [ ] Implement proper password complexity requirements and validation

### Input Validation & Sanitization
- [ ] Add comprehensive input sanitization for file system operations (prevent path traversal)
- [ ] Implement file upload validation (file type whitelist, size limits, virus scanning)
- [ ] Add SQL injection protection (using parameterized queries, already using Mongoose but verify)
- [ ] Implement XSS protection for user-generated content (terminal output, file names, etc.)
- [ ] Add Content Security Policy headers (partially done, needs refinement)
- [ ] Validate and sanitize plugin code before execution (sandboxing)
- [ ] Implement file system quota limits per user
- [ ] Add validation for workspace names, file names (length, special chars, etc.)
